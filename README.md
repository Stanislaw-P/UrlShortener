# UrlShortener

Высокопроизводительный микросервисный сервис сокращения ссылок (аналог Bitly), построенный на платформе .NET 8 с асинхронной обработкой аналитики переходов.

## 🚀 Архитектура и технологии
Проект спроектирован с упором на отказоустойчивость, высокую скорость чтения и изоляцию нагрузок:

### Компоненты системы:

UrlShortener.API: Веб-служба, отвечающая за генерацию коротких кодов и обеспечивающая мгновенный редирект пользователей.

UrlShortener.Analytics (Worker): Изолированный фоновый сервис, который асинхронно обрабатывает статистику кликов, не замедляя основной поток редиректа.

### Стек технологий:

 - База данных: PostgreSQL (хранение ссылок и статистики переходов).
 - Кэширование: Redis (микросекундный редирект за счет прогрева кэша при создании/первом поиске ссылки).
 - Брокер сообщений: RabbitMQ + MassTransit (асинхронная доставка событий клика от API к Воркеру аналитики).
 - Алгоритм кодирования: Чистый Base62 на основе автоинкрементных ID для генерации красивых коротких URL.

## 🛠️ Инструкция по локальному запуску (Docker Compose)
Вся инфраструктура и приложения полностью контейнеризированы. Для запуска всей системы «из коробки» вам понадобятся только установленные Docker и Docker Compose.

### 1. Клонирование репозитория
```
git clone https://github.com/Stanislaw-P/UrlShortener.git
cd UrlShortener
```
### 2. Сборка и запуск контейнеров
Выполните команду в корневой папке решения (там, где находится файл docker-compose.yml):
```
docker-compose up --build -d
```
### 3. Доступные эндпоинты системы
После успешного запуска инфраструктура и сервисы будут доступны по следующим адресам:
| Сервис | URL |
|---|---|
| Swagger (Документация API)| http://localhost:8080/swagger или http://localhost:8080/swagger |
| RabbitMQ UI | http://localhost:15672 (guest / guest) |
| База данных Postgres | localhost:5432 База: url_shortener_db, Пользователь: postgres, Пароль: postgres_pass |
| Кэш Redis: | localhost:6379 |

## 🚦 Проверка работоспособности (Quick Start)
1. Откройте Swagger UI и отправьте ```POST``` запрос на /api/urls/shorten:
```
{
  "longUrl": "https://github.com/phatboyg/MassTransit/123"
}
```
2. Скопируйте полученный код и выполните ```GET``` запрос редиректа: ```http://localhost:8080/{code}```
3. Система мгновенно перенаправит вас на оригинальный URL.
4. Откройте логи контейнера ```shortener-analytics``` или загляните в таблицу ```LinkStats``` в PostgreSQL — вы увидите, что воркер асинхронно поймал событие клика, зафиксировал ваш IP-адрес, увеличил счетчик переходов и обновил время последнего клика.

## 🛑 Остановка проекта
Чтобы остановить систему и освободить порты, выполните:
```
docker-compose down
```
Если вы хотите полностью очистить данные баз данных и очередей для чистой повторной сборки, выполните: ```docker-compose down -v```
